# Module 1 - Design Patterns

## SOLID Principles
- SRP - Single Responsibility Principle
- OCP - Open/Closed Principle
- LSP - Liskov Substitution Principle
- ISP - Interface Segregation Principle
- DIP - Dependency Inversion Principle

## Creational Patterns
- Singleton - Ensures only one instance of a class exists.
- Factory Method - Creates objects without exposing creation logic.
- Builder - Builds complex objects step by step.

## Structural Patterns
- Adapter - Makes incompatible interfaces work together.
- Decorator - Adds new functionality dynamically.
- Proxy - Controls access to an object.

## Behavioral Patterns
- Observer - Notifies dependent objects when state changes.
- Strategy - Allows switching between algorithms at runtime.
- Command - Encapsulates a request as an object.
